Explored both codebases' worktree and merge code in parallel using subagents.

- Dry-run mode

**Foreman gaps identified:**
- Binary conflict resolution (auto-resolve reports OR create PR)
- No persistent queue
- Force-deletes branches without merge-status awareness

## Key Decisions

| # | Decision | Rationale |
|---|----------|-----------|
| 1 | Hybrid auto-enqueue with reconciliation | Safety net catches race conditions without manual intervention |
| 2 | Sonnet for Tier 3, Opus for Tier 4 | Cost-effective tiering — Sonnet handles structured resolution, Opus handles complex multi-file conflicts |
| 3 | Per-file tier cascade (not whole-merge) | Avoids escalating entire merge when only one file needs AI |
| 4 | Graph-based conflict clustering | Optimizes merge ordering to minimize conflicts |
| 5 | git reset --hard HEAD~1 (not git revert) | Cleaner rollback — no revert commit noise in history |
| 6 | messages.create() from @anthropic-ai/sdk | Direct API control for merge resolution, avoids Agent SDK overhead |
| 7 | No --legacy flag | Queue is the only path from day one — simplifies codebase |
| 8 | Tests only after AI merges (Tier 3/4) | Deterministic merges (Tier 1/2) are safe; testing AI output catches hallucinations |
| 9 | Refinery preserved as thin wrapper | Maintains existing architecture pattern for AI calls |
| 10 | Sprint 3 split at quality gate | Ensures deterministic resolution is solid before introducing AI |
